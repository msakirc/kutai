## The substrate you now have (use it; don't reinvent)

- **husam** (`packages/husam/`, kumandan hüsamettin) — the non-agentic single-call worker. The orchestrator pump routes `raw_dispatch` tasks (`context.llm_call.raw_dispatch=True`) → `husam.run(task)` → select→execute→map. A mechanical/tool can also call `husam.run(spec)` **inline** for a synchronous LLM result (the critic_gate `produce_verdict` precedent — **shape-(a)**). Cleaner is **shape-(b)**: enqueue an admitted task → pump → husam → result via CPS continuation, mechanical just confirms.
- **dumb-pipe dispatcher** — `execute(pick, messages)` only. Workers select (`fatih_hoca.select`) + map. Don't call `dispatcher.request` (shopping-only shim now).
- **post-hook CPS** — `_enqueue_posthook_llm_child` spawns a raw_dispatch child + `on_complete`/`on_error` continuation → `_apply_posthook_verdict`. Result-rewrite verdict (`PostHookVerdict.action="rewrite"`) + ordered chain (`_advance_posthook_chain`, `_posthook_queue`). Per-source verdict lock + `reconcile_stranded_posthook_chains`.
- **durable continuations** (SP1) — `enqueue(on_complete=, on_error=, cont_state=)`, claim-then-fire CAS, restart-reconcile.

## Landmines from the SP3b review — SP4 MUST heed these

- **NEVER `lane="overhead"` (or any lane ∉ {`oneshot`,`ongoing`}).** It's a phantom lane the pump never selects → silent orphaning. `add_task` persists it verbatim (no validation). Use `lane="oneshot"` for overhead LLM children (the pump only pumps oneshot; there is no separate overhead/ongoing pump from the orchestrator). This bit SP3 + SP2 + yalayut; all fixed — don't reintroduce.
- **Mechanicals must not call the dispatcher directly** (`[[feedback_no_direct_dispatcher]]`). `husam.run`-inline from a mechanical (shape-a) removes the *dispatcher* call but the mechanical still reaches a worker un-admitted — acceptable as a stopgap (critic_gate did it), but **shape-(b)** (separate admitted step + mechanical confirm) is the sanctioned end state. `critic_gate.confirm_gate` exists as the unused shape-b seam — decide the shape for SP4's mechanical LLM gates (founder call).
- **husam runtime install:** any NEW `import husam` site needs `pip install -e packages/husam` to be live. Note it in the SP4 "done" criteria.
- **Concurrency in tests:** NEVER run two pytest invocations at once — concurrent pytest deadlocks the shared SQLite DB and crash-loops the live KutAI (a verify-subagent did this during SP3b → "hung test"; recovery = kill only the pytest PIDs, never llama-server/orchestrator/wrapper). Run suites **sequentially**, one `timeout`-prefixed invocation at a time.
- **Worktree conftest is a HARDCODED package list, not a glob** (`conftest.py` `_PACKAGE_SRCS` + the eviction set). Any NEW package SP4 adds must be appended to BOTH, or its tests won't resolve it.
- **`_task_result_to_request_response` stays stable** — consumers: grading, brand_voice_lint, vision. If SP4 migrates vision/brand_voice off it, update/retire it carefully.
