# SP3b — Dumb-pipe dispatcher + hüsam non-agentic worker — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the LLM dispatcher a dumb pipe (one `execute()` interface, no logic), stand up `husam` (display: *kumandan hüsamettin*) as the non-agentic single-call worker, move coulson's `reflection` + `constrained_emit` to result-rewriting post-hook tasks, and split `critic_gate` — killing the `request()→enqueue(await_inline)` deadlock for every in-scope caller.

**Architecture:** Beckman admits every task (sole gate). The orchestrator pump routes each admitted task to a worker — `coulson` (react), `husam` (raw single calls), `mr_roboto` (mechanical). Workers select (`fatih_hoca.select`) and call the dispatcher's one primitive `execute(pick, messages)`. No worker imports another. reflection/emit leave coulson and become post-hook tasks that run on husam and **rewrite** the source result before `grade`.

**Tech Stack:** Python 3.10, asyncio, pytest (always `timeout`-prefixed), SQLite (aiosqlite), the existing packages: `general_beckman`, `coulson`, `fatih_hoca`, `mr_roboto`, `hallederiz_kadir`. New package: `husam`.
