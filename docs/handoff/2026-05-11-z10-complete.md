## What changed

### Phase A — Reversibility + provenance + plug min_confidence
- `confidence_gate` field on agents (`fail_closed` default; Researcher = `warn` opt-out). **The gate was cosmetic for 8+ months — now fail-closed real.**
- `file_locks` orphan sweeper (60s cadence) + `expires_at` column.
- Atomic `git_commit ↔ git_push` (opt-in `payload.push=True`; orphan rollback default, `allow_orphan=True` override).
- `VERB_REVERSIBILITY` registry — 69 verbs tagged (49 full / 3 partial / 6 irreversible + 9 added by later tiers). Cross-check test fails loudly when new verb lacks a tag.
- `Action.reversibility` populated by dispatcher from registry; per-invocation override via `reversibility_override` payload.
- `schema_migrations(version, applied_at, sql, reversal_sql, description)` ledger + `apply_migration()` helper. **Absorbed T1A's `_migrations_pending.txt` breadcrumb** — every Z10 DDL goes through it now (15 migrations applied).
- `tasks.confidence_categorical / confidence_numeric / reasoning / reversibility` columns.
- `artifact_provenance(path, task_id, step_id, model_id, retry_n, reviewer_verdict_id, mission_id, written_at)` table + `db.get_artifact_provenance(path)` joins to tasks + model_call_tokens + reviewer.
- `registry_events` extended (mission_id, task_id, verb, reversibility) — per-action audit. Mr. Roboto dispatcher logs every Action.
- `action_confirmations` table + `request_confirmation / check_confirmation / resolve_confirmation` skeleton (poll-with-timeout; T2B replaced poll with reaction-driven resolution).

### Phase B — Cost transparency

- `model_call_tokens.cost_usd` per-call (computed from model rate × tokens; HaLLederiz Kadir writes it).
- `cost_by_iteration` view — **first-pass vs retry tax visible** (was uncomputable on main).
- `mission_budget_alerts(threshold, total_usd, posted_at, drained_at)` + 5min cron threshold-cross detection (50/75/90%). UNIQUE on `(mission_id, threshold)` = idempotent.
- `tasks.estimated_cost_usd` + `actual_cost_usd` (Beckman writes estimate on dispatch; finalize on complete).
- `missions.cost_decision_threshold_usd` + `request_confirmation(verb='cost_decision')` when multi-pass action exceeds threshold.
- `missions.quality_mode` (`quick|balanced|thorough` default `balanced`) → coulson retry caps + reviewer rounds + fatih_hoca weights.
- `record_vendor_cost` in `kuleden_donen_var.cost_adapter` for vendor adapters.
- `format_mission_cost(mission_id)` formatter + `/mission_cost` Telegram command.

### Phase E — Sandbox + concurrency safety
- Per-mission Docker container `kutai-mission-{id}` via `ensure_mission_container` / `teardown_mission_container`. Hooked into `add_mission` + workflow_engine `_maybe_complete_mission`. Resource caps 4g/2cpu/512pid (env override + `missions.sandbox_resource_overrides_json` per-mission).
- `missions.sandbox_mode` default `docker`; mission-requested `local` opens `request_confirmation(verb='sandbox_local_mode')`. **The silent `SANDBOX_MODE=local` escape is closed.**
- Semantic argv guard `_is_blocked_argv` — denies `dd / mkfs / fdisk / parted / wipefs / shred / iptables / nft / route / ip link|route / nc -l / ncat -l / socat ...LISTEN` + path args matching `/dev/sd*` `/dev/nvme*` `/dev/disk*` `/proc/sysrq-trigger` `/sys/kernel*`. Splits on `;`/`&&`/`||`/`|` for post-separator args. Additive to existing regex set. **Catches `["dd", "if=/dev/sda", ...]` argv tricks (was the documented escape).**
- `config/egress_allowlist.txt` + `request_egress_to` shell kwarg → `request_confirmation(verb='broader_egress')` for off-whitelist hosts. Whitelist seeded with vendor hosts (api.openai.com, api.anthropic.com, api.groq.com, openrouter.ai, huggingface.co, pypi.org, files.pythonhosted.org, github.com, raw.githubusercontent.com, objects.githubusercontent.com, registry.npmjs.org).
- **`_tx_lock` sharded per-mission** via `_get_tx_lock(mission_id)` + `_get_combined_lock` (global-then-mission, deadlock-safe). Mission-scoped tables: `tasks`, `task_events`, `mission_events`, `artifact_provenance`, `mission_pacing_snapshots`, `mission_tradeoff_prompts`, `mission_budget_alerts`, `action_confirmations`, `cost_budgets WHERE scope='mission'`, `mission_green_tags`. Global: `missions` (row), `models`, `model_pick_log`, `model_call_tokens`, `registry_events`, `schema_migrations`, `cost_budgets WHERE scope IN ('vendor:*','global')`.
- **Per-mission Chroma namespace** — collections renamed `mission_{id}__<base>` (per-mission) or `global__<base>` (skills). `mission_chroma_collection_name(mission_id, base)` helper. One-shot migration script `scripts/migrate_chroma_to_per_mission.py` (dry-run default; `--apply` to commit).
- **Reset-to-green primitive** — `mr_roboto.mark_green(mission_id, task_id)` triggered explicitly via `git_commit` step `payload.mark_green=True` (documented contract — not auto-fired). Tags workspace git `green-{mid}-{tid}`, snapshots mission DB rows + Chroma collections to `data/snapshots/mission-{mid}-{tid}/`, appends to `mission_green_tags`. `rollback_mission(mission_id, target_task_id?)` checks out tag → walks `schema_migrations` newer than green and applies `reversal_sql` in reverse (best-effort; NULL `reversal_sql` skipped + WARNING) → restores mission rows + Chroma collections. `/rollback_mission <id> [task_id]` Telegram command. `rollback_mission` tagged `irreversible` → T1C confirmation auto-gates.

### Phase G — Trust calibration loop
- `confidence_outcomes(task_id, mission_id, agent_type, task_kind, model_id, picked_at, confidence_*, outcome_correct, outcome_resolved_at, resolution_source, reviewer_verdict_id, notes)`. Partial index `WHERE outcome_correct IS NULL` for unresolved.
- Wire site: `general_beckman/apply.py::_apply_code_review_verdict` → `_record_and_resolve_confidence(task_id, correct=verdict_pass, source='reviewer_verdict')` on both pass + fail exits. No-op when source task has no confidence signal. `task_kind = tasks.agent_type` (v1 proxy).
- `confidence_reliability_scores(model_id, task_kind, confidence_bucket, sample_n, correct_n, reliability, updated_at)` — 6h `confidence_calibration_recompute` cron rolls outcomes into this table.
- `coulson.context.build_system_prompt` consults cache populated by `refresh_calibration_cache`. For picked (model_id, task_kind, 'high'): `reliability < 0.65 + sample_n ≥ 30` → conservative note; `> 0.85` → trusted note; else no injection. Env tunable: `CALIBRATION_MIN_SAMPLE_N=30`, `CALIBRATION_LOW_THRESHOLD=0.65`, `CALIBRATION_HIGH_THRESHOLD=0.85`.

## Known gaps / follow-ups

1. **`model_pick_log.task_id`** — currently matches by `task_name`; tighter attribution in a future tier.
2. **Egress firmness** — whitelist is gate-by-prompt v1; no in-container iptables drop yet. Full enforcement needs netns/firecracker (post-v1, called out in 10-cross-cutting.md).
3. **Calibration cache process-local** — cron refresh covers in-process freshness; multi-process deploy needs a shared invalidation channel.
4. **`mark_green` Chroma snapshot** — JSON serialization slow + RAM-heavy for >10k embeddings. Optimization route: `shutil.copytree` per-mission Chroma sub-directories (needs `PersistentClient` per mission, larger refactor).
5. **`record_demo` runtime assumption** — Playwright + ffmpeg presumed in mission container image. Operator-side responsibility; not enforced at workflow-expand time.
6. **`MISSION_SCOPED_TABLES` hand-curated** — new mission-scoped tables in future tiers must be added to the list (otherwise rollback misses them).
7. **`skip_when: no_e2e_specs` token informational** — runtime evaluator doesn't auto-resolve list tokens yet; `record_demo` does its own check internally for v1.
8. **5 pre-existing `pick_log` failures + 2 clarify/notify_user** — preceded Z10, NOT introduced by this campaign. Triage in a future cleanup tier.
9. **i2p_v3.json `15.10b/c` and `15.14b`** — uses inserted IDs to avoid renumber churn; eventual renumber pass would be cleaner.
