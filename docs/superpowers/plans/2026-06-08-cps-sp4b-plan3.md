**Goal:** Remove the 3 Plan-3 `await_inline=True` call sites (crisis/draft_holding, incident/draft_update, press_kit/assemble) by moving each blocking LLM hop onto the durable CPS continuation substrate, so SP5 can delete the primitive.

**Architecture:** Mirror Plan 1's reviews migration exactly. Each verb's LLM call becomes an admitted Beckman producer (`enqueue(..., on_complete=, on_error=, cont_state=)`, NO `await_inline`); a mechanical continuation handler (the "sink") does the parse + canned fallback + founder-facing side-effect. Prompts live OUTSIDE mr_roboto (in `src/comms/producers.py`). Sinks live in `mr_roboto/executors/` and are registered in `continuations._HANDLER_MODULES` for restart-recovery. **No General Beckman core change. No `degrade_on_exhaustion`.**

**Tech Stack:** Python 3.10 async, General Beckman continuation substrate (`packages/general_beckman/src/general_beckman/continuations.py`), pytest.

**Context — these 3 verbs are DEAD (built, never triggered).** `/crisis open` (`telegram_bot.py:5704`) opens an event and points the founder at the playbook; it does NOT enqueue `crisis/draft_holding`. No code enqueues `incident/draft_update` or `press_kit/assemble` either (verified grep: only verb modules, router branches at `mr_roboto/__init__.py` 4273/4414/4536, posthook comments, and tests reference them). So there is **no live trigger to repoint** beyond the router branches, and **no live integration test** to write. The migration makes the producer the future entry point; router branches are repointed to enqueue it so the action stays dispatchable via CPS.

async def enqueue_crisis_holding(*, event_id: int, product_id: str, tier: int,
                                 summary: str, playbook_excerpt: str = "") -> int | None:
    """Enqueue the crisis holding-statement producer; comms.crisis_holding.resume

    tier = int(tier or 1)
    label = _CRISIS_TIER_LABELS.get(tier, f"tier {tier}")
    excerpt = (playbook_excerpt or "")[:1200] or "(playbook not available)"
    prompt = (
        f"You are drafting holding statements for a Tier {tier} crisis ({label}).\n\n"
        f"Crisis summary: {summary or 'Crisis event opened.'}\n\n"
        f"Playbook context (holding statement shape section):\n{excerpt}\n\n"
        "Produce EXACTLY 2 holding-statement variants:\n"
        "- Variant A: More formal / corporate tone.\n"
        "- Variant B: More human / conversational tone.\n\n"
        "Rules:\n- Each variant: 2-4 sentences. Under 280 characters.\n"
        "- No internal hostnames, team names, or technical jargon.\n"
        "- No speculation about root cause.\n"
        "- Tier 3+: do NOT confirm 'breach' — use 'incident' or 'security event'.\n"
        "- Tier 4: extremely minimal — confirm aware and investigating; NO details.\n\n"
        'Return ONLY a JSON array of 2 strings: ["Variant A text", "Variant B text"]'
    )

        on_error="comms.crisis_holding.resume_err",
