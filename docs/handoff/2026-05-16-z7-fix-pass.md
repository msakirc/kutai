### SYSTEMIC A — broken LLM-enqueue `_callback` pattern (5 sites)

**Symptom:** verb enqueues a task with an async function in `context["_callback"]` + a
`done_event`, then `await asyncio.wait_for(done_event.wait(), ...)`. `general_beckman.enqueue`
with `await_inline=False` never invokes callbacks, and `json.dumps(context)` in `add_task`
chokes on the coroutine → `TypeError` → swallowed by the outer try/except → silent fallback
to canned/heuristic output. Every shipped output is fake.

**Correct pattern:** `enqueue(spec, await_inline=True)` returns a `TaskResult` synchronously.
Working reference implementations: `packages/mr_roboto/src/mr_roboto/demo_storyboard.py`,
`press_kit_assemble.py`.

**Sites to fix:**
- `packages/mr_roboto/src/mr_roboto/incident_draft_update.py:159-180`

### STANDALONE CRITICALS

1. `src/app/attention_budget.py:84-87` — `_fetch_cards` does `await cur.close()` then reads
   `cur.description`. aiosqlite nulls/raises after close → **A0 attention-queue render
   crashes**. Read `description` before closing.
2. `src/app/webhook_listener.py:34,148-153` — `logger = logging.getLogger("kutai.webhook")`
   (stdlib) but the email-webhook handler calls `logger.error("...", provider=..., product_id=...,
   exc=...)`. Stdlib rejects kwargs → `TypeError` on every email-webhook error path. Switch to
   `get_logger`.
3. B3 review gate never fires — `apply.py:1631-1645` builds the `incident_update_review`
   payload from `source_ctx.get("draft")` (task context), but `incident/draft_update` returns
   the draft in its **result dict** (`incident_draft_update.py:269-276`), never writing it to
   context. Handler hits `if not draft: skip`. Status updates publish **unreviewed**. Fix:
   write `draft` into task context, or change `apply.py` to read from the result.
4. `incident_publish_status.py` never calls `status_page.invalidate_cache()` despite the
   docstring claiming it does. `/status` serves stale content ~60s after publish. Wire it.
5. A7 warmup quota bypassable — `outreach_send.py:138-164` enforces quota only
   `if warmup_row is not None`. A fresh domain (no row) sends with zero throttle. Treat a
   missing row as day-1 quota / seed the row.
6. A11 Twitter gate dead — `mention_polls.py:402`:
   `if not os.getenv("MENTION_TWITTER_ENABLED","").strip() == "1":` — `not` binds tighter than
   `==`, so the early-return never fires. Fix: `if os.getenv("MENTION_TWITTER_ENABLED","").strip()
   != "1": return []`.
7. B9 audit log not wired — `audit_log.py:295` `wrap_external_verb` keys on `result.get("ok")`
   but executors return `Action`/dicts with `status`; `@wrap_external_verb` is applied to ZERO
   verbs; `EXTERNAL_PUBLISH_VERBS` is unused. Also no hourly `audit_completeness_check` cron in
   `cron_seed.py`. Fix: correct the result key, actually decorate (or explicitly call
   `log_external_send` in) the external-publish verbs, add the hourly cron.
8. `mentions` UNIQUE not product-scoped — migration `2026-05-16-z7-a11-mentions` uses
   `UNIQUE(source, source_id)`; two products mentioned in the same HN/Reddit thread collide.
   Should be `UNIQUE(product_id, source, source_id)`. (New migration — don't edit the applied one;
   add a corrective migration, or if no real data exists, recreate the table.)
9. B1 announcement blast broken — `changelog_publish.py:62` passes `user_id=None` into
   `email_sends.user_id` (NOT NULL); no subscriber fan-out exists. The "email blast" surface is
   non-functional. Implement broadcast fan-out over subscribed `email_preferences`.
10. B4 `meeting/brief` ships a stub — `packages/mr_roboto/src/mr_roboto/__init__.py:3556-3608`
    never calls an LLM; every brief contains the literal placeholder
    `"(LLM-drafted talking points will appear here...)"`. Wire the LLM (await_inline) for
    talking points + suggested asks.
11. B7 missing consent gate — `src/app/interview.py:124-164` `transcribe_interview` transcribes
    customer audio with NO `interview_recording` consent check, though `db.py:3364` comment says

12. B1 send job ignores preferences — `src/app/jobs/lifecycle_email_send.py` calls `send_email`
