## Critical unwired

Items shipped as features that have **zero** live call site (Telegram-only or workflow-only invocation paths missing).

- **`index_idea_fingerprint` is never enqueued anywhere.** Function exists at
  `packages/mr_roboto/src/mr_roboto/find_similar_missions.py:322` and the
  mechanical dispatch handler at `packages/mr_roboto/src/mr_roboto/__init__.py:2196`
  is in place, but no code path calls it. T6A handoff said
  "caller must explicitly dispatch after Continue/new-mission confirm" — that
  caller does not exist. Consequence: the `mission_ideas` ChromaDB collection
  used by `find_similar_missions` will stay empty forever; the
  `legacy_pre_idea_dedup` column (db.py) has no `skip_when` consumer either.
  Suggested fix: enqueue after the founder's accept-callback in
  `telegram_bot.py` callback handler, OR add a post-hook on 0.6 alongside
  `find_similar_missions`.

- **`_edit_html_upload` pending action stash is never consumed.**
  `cmd_edit_html` at `src/app/telegram_bot.py:4027` stashes
  `_pending_action[chat_id] = {"command": "_edit_html_upload", ...}` but
  grep of the bot shows zero handlers checking that command tag (only the
  stash line itself). No `MessageHandler(filters.Document...)` is registered
  in `_setup_handlers`. Net: `/edit_html` prompts the founder for a document,
  founder uploads, nothing happens. `propose_spec_patch_from_html_diff` exists
  at `packages/mr_roboto/src/mr_roboto/propose_spec_patch.py:174` and is
  dispatch-callable, but with no upload consumer it can never run live.
  T4B handoff acknowledged this as deferred.

- **`critic_log`, `regen_log`, `preview_log`, `paraflow_diff_log` write-only.**
  Writers exist at:
    - `packages/mr_roboto/src/mr_roboto/critic_gate.py:154`
    - `packages/mr_roboto/src/mr_roboto/regen.py:210`
    - `packages/mr_roboto/src/mr_roboto/emit_preview_url.py:236`
    - `packages/mr_roboto/src/mr_roboto/verify_against_paraflow_goldens.py:103`
  But no `FROM critic_log|regen_log|preview_log|paraflow_diff_log` SELECT
  in production code (grep returned zero hits outside tests). Telemetry is
  forward-only with no surfacing UI/job. Not catastrophic — audit logs are
  legitimate — but if these were meant to drive any UI ("show last 5 regens"
  / "list previews / diff trend"), that consumer is missing.

- **`streaming_guard_log` table created, NO writer at all.**
  `src/infra/db.py:859` creates the table; `coulson/streaming_guards.py:10`
  docstring mentions "writes to streaming_guard_log" but the only outcome
  surface in the pipeline is `logger.warning`. Handoff T5C flagged this:
  "table created but no DB writer wired yet — outcomes surface only via
  `logger.warning`. Add writer as small follow-up." Confirmed: zero
  `INSERT INTO streaming_guard_log` anywhere in `src/` or `packages/`.

## Shallow scaffolds

Bodies that don't deliver what the name implies.

- **`regen._invoke_emitter` fallback ≈ no-op.**
  `packages/mr_roboto/src/mr_roboto/regen.py:128-183`: when
  `coulson.execute` is unavailable, the fallback returns
  `"<!-- regen: <change> -->\n<!-- axis: <axis> -->\n" + current_body` —
  i.e. it prepends an HTML comment and returns the **unchanged** body.
  Also, if `coulson` is available but raises, line 179 returns
  `{"text": current_body}` (the original, untouched). For a "regen"
  feature this is materially misleading when the LLM path fails: caller
  sees a successful response and writes the same content with a comment
  tag. T4A handoff acknowledged this as a deterministic shim. Production
  must monitor logs for "coulson failed" or `regen_log` rows whose
  `delta_chars == 0`.

- **`run_bash_audit` cron payload uses `_executor` not `action`.**
  `packages/general_beckman/src/general_beckman/cron_seed.py:142` has
  `"payload": {"_executor": "run_bash_audit"}`. mr_roboto dispatch
  (`__init__.py:309` onwards) keys on `payload["action"]`, not
  `payload["_executor"]`. Need to verify Beckman's mechanical-task router
  translates `_executor` → `action` before dispatch. If it doesn't, this
  cron will never execute the audit. (Did not trace Beckman router
  end-to-end; flag for verification.)

- **Compliance templates exist but several are skeletal.** All 10 `.md.j2`
  templates under `compliance_templates/` are real Jinja2 with conditionals
  + sections (50-111 lines each). They are not stubs in the sense of TODO
  placeholders, but they ARE labelled "review with counsel before
  publishing" and have lines like "Data categories not yet declared.
  Founder must complete the compliance fingerprint before this section is
  binding." Adequate as a scaffold; not adequate as a legal document.
  Handoff T5A acknowledged: "Templates beyond `default/en/privacy_policy.md.j2`
  are stubs — hand-curate per spec." Confirmed: they ARE real, just
  minimal. Not flagged as a critical scaffold.

## Wiring gaps (smaller)

- **`legacy_pre_critic_gate` column created, no `skip_when` consumer.**
  `src/infra/db.py:822` adds the column; zero references in `i2p_v3.json`.
  Because `critic_gate` fires inline on git_commit/notify_user (not as
  a workflow step), there's nothing to gate. Either the column should be
  dropped, or the gate logic in `mr_roboto/__init__.py:325` should consult
  it before invoking `_critic_gate`. Currently it consults env+payload only
  (`_critic_opt_out()` env + `payload.get("critic_gate", True)`).

- **`legacy_pre_idea_dedup` column created, no `skip_when` consumer.**
  `src/infra/db.py` adds the column; no step uses it. Same root cause as
  the index_idea_fingerprint unwiring above.

- **`legacy_pre_design_tokens` column created, no `skip_when` consumer.**
  Added at `src/infra/db.py:377`, zero refs in `i2p_v3.json`. Either dead
  column or a planned T3 gate that didn't land.

- **`legacy_pre_user_flow` column created, no `skip_when` consumer.**
  Added at `src/infra/db.py:388`, zero refs in `i2p_v3.json`. Same as above.

- **`researcher` agent prompt does NOT mention `find_prior_art`.**
  `src/agents/researcher.py:25` lists `allowed_tools` = `web_search`,
  `read_file`, `write_file`, `file_tree`, `api_lookup`, `api_call`,
  `play_store`, `github`, `pharmacy`. No `find_prior_art`. The tool IS
  registered globally in `src/tools/__init__.py:96` as an
  `_optional_tools` entry. The step 1.0 instruction itself names the
  function ("Call find_prior_art(...)"), so the agent CAN call it via
  tool_call dispatch even without prompt mention. But the agent prompt
  also says "Use `web_search` once" — and `web_search` is listed by name
  while `find_prior_art` is not. Risk: the agent under-uses
  `find_prior_art` because its system prompt teaches `web_search`-only.
  Confirmed allowed_tools omission is real.

- **Root `conftest.py` line 19 sets `KUTAI_CRITIC_GATE=off` via
  `os.environ.setdefault`.** This is the PRODUCTION conftest, not a
  worktree leftover (handoff T5C mentioned worktree-root). Production
  doesn't run via pytest, so the import is harmless — but
  `os.environ.setdefault` runs whenever pytest is invoked at the
  repo root. If a future cron or CI hook ever shells into the worktree
  via `pytest`, this could leak. Comment at line 18 explicitly calls out
  "Production must keep the var unset" — correct, but worth verifying
  no orchestrator startup path imports `conftest.py`.

- **Step 5.40 `emit_preview_url` `depends_on` 5.30c not 5.30b.verify_shape.**
  Documented in T4C handoff as intentional. Note for future readers.

## Cross-cutting recommendations

1. **Wire `index_idea_fingerprint` to the founder-accept callback** so
   `mission_ideas` collection actually grows. Without it T6A's core value
   (cross-mission idea dedup) doesn't accrue.

2. **Decide on `_edit_html_upload`** — either land the document-upload
   handler in `telegram_bot.py`, or remove the stash in `cmd_edit_html` so
   it doesn't silently lie to the founder.

3. **Add `streaming_guard_log` DB writer** in the streaming-guard outcome
   path inside `caller.py:374` (warn/fix/halt branches each insert).

4. **Audit Beckman's mechanical-task router** for `payload._executor` vs
   `payload.action` translation; the `run_bash_audit` cron will silently
   no-op if that translation is missing.

5. **Prune dead `legacy_pre_*` columns** (`critic_gate`, `idea_dedup`,
   `design_tokens`, `user_flow`) or wire `skip_when` consumers — current
   state misleads anyone reading the schema as "these gates are active".

6. **Add `find_prior_art` to `researcher.allowed_tools`** + a one-liner
   in the system prompt so the agent treats it as a peer to `web_search`.

7. **Move root `KUTAI_CRITIC_GATE=off` setdefault out of `conftest.py`
   into a `tests/conftest.py` or `pytest` env-fixture** — the current
   placement runs on any python -m pytest from the project root, and the
   "production must keep unset" instruction is comment-only enforcement.
